# CiA Assistant — Chrome Extension

A Manifest V3 Chrome extension that surfaces the CiA Transition Assistant in a side panel and via a right-click context menu, so the assistant follows you across every tab.

## What's included

- **Floating widget** (Copilot-style) — a small magenta bubble in the bottom-right of every page. Click it and a draggable, resizable chat panel pops up over the current page. Embeds the same React side-panel UI via an iframe.
- **Side Panel** (browser-native) — used as a fallback on restricted pages (`chrome://`, the Chrome Web Store, etc.) where content scripts can't run. Available anytime via the dock button (⇲) inside the floating panel.
- **Toolbar action** — clicks toggle the floating widget (or open the side panel on restricted pages).
- **Context menu** — select any text on a page, right-click, choose "Ask CiA about ..." to prefill the chat composer.
- **Page-context chip** — automatically attaches the current tab's URL, title, and any selected text to your message (toggleable).
- **Options page** — configure the API base URL and test the connection.
- **Login fallback** — if the cookie-based session from the web client isn't present, sign in directly inside the extension.

## Develop

From the project root:

```bash
npm install
npm run dev:extension     # runs server + extension Vite dev server in parallel
```

Then load the unpacked extension in Chrome:

1. Open `chrome://extensions`.
2. Toggle **Developer mode** (top right).
3. Click **Load unpacked** and select `extension/dist`.
4. Pin the extension and click its toolbar icon — the side panel opens.

The dev server emits `dist/` continuously; Chrome auto-reloads when files change.

## Build for distribution

```bash
npm run build:extension   # outputs extension/dist
```

Zip the contents of `extension/dist` for upload to the Chrome Web Store, or distribute as an unpacked load for internal testing.

## Configure the API + web-app URLs

Right-click the extension icon → **Options** (or open `chrome://extensions` → Details → Extension options). The options page configures:

- **API base URL** — where the Express backend is running. Default `http://localhost:3001`. Used for chat, conversations, auth. There's a **Test connection** button that pings `/health`.
- **Web app URL** — where the full React web app is hosted. Default `http://localhost:5173`. Opened by every "↗ Open web app" button (in the side panel header, the floating widget header, and the options page's prominent CTA).

If you point the API at a non-localhost host, the extension will request optional `host_permissions` for that origin so it can call the API with credentials.

## Notes on auth & cookies

- The extension calls the API with `credentials: 'include'`. The server's CORS allow-list now accepts `chrome-extension://` and `moz-extension://` origins automatically.
- For production (HTTPS) deployments, ensure `COOKIE_SECURE=true` and consider setting `SameSite=None` on the session cookie so it survives extension → server cross-site requests.
- During local dev (HTTP localhost), Chrome generally permits the cookie round-trip without further configuration.

## Project layout

```
extension/
├── manifest.config.js     # MV3 manifest, generated by @crxjs at build time
├── public/icons/          # generated PNGs (16/32/48/128) — do not commit
├── scripts/
│   └── generate-icons.mjs # zero-dep PNG icon generator (runs pre-build)
└── src/
    ├── background/        # service worker (action click, context menu, prefill bus)
    ├── content/           # content script (page context provider)
    ├── lib/
    │   ├── api.js         # REST + SSE client (mirrors client/src/lib/api.js)
    │   ├── pageContext.js # active-tab page context grabber
    │   └── storage.js     # chrome.storage wrapper, API URL persistence
    ├── options/           # options page (React)
    ├── sidepanel/         # side panel (React)
    └── styles/extension.css
```
